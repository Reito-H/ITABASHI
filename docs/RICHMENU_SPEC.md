# LINEリッチメニュー 仕様書

**作成日**: 2026-06-17  
**ステータス**: 設計中（パターン1〜3は実装予定・パターン4は将来実装）

---

## 1. 概要

LINE公式アカウント「ITABASHI」のリッチメニューを、連携ユーザーの種別に応じて自動的に切り替える機能。  
ユーザーがメッセージを送信したタイミング、または連携完了タイミングで、該当するリッチメニューをLINE APIで紐付ける。

### ユーザー種別と対応パターン

| ユーザー種別 | パターン | ステータス |
|---|---|---|
| 未連携（デフォルト） | パターン3 | 実装予定 |
| 新人社員 | パターン1 | 実装予定 |
| 班長・指導者 / 車番検索管理者 | パターン2 | 実装予定 |
| 一般乗務員（将来） | パターン4 | 未実装・将来対応 |

## 2. パターン定義

### パターン3 — 未連携ユーザー（デフォルトメニュー）

**対象**: LINE友達追加済みだが、まだ誰とも連携していないユーザー  
**レイアウト**: 1行 × 1列（ボタン1つ）  
**推奨サイズ**: 2500 × 843px（コンパクト）

| ボタン | ラベル | アクション |
|---|---|---|
| （全面） | LINEを連携する | テキスト送信「車番連携」 or 固定メッセージ案内 |

**補足**: このメニューはLINE OA Managerで「デフォルトリッチメニュー」として設定する。連携完了後は個別割り当てにより上書きされる。

**推奨サイズ**: 2500 × 1686px（フル）

| ボタン | ラベル | アクション |
|---|---|---|
| 既存のレイアウト・内容に準ずる | — | — |

**連携タイミング**: 招待コード入力 → 紐付け成功時に自動割り当て

## 3. 技術仕様

### 使用するLINE API

| 操作 | エンドポイント |
|---|---|
| リッチメニュー作成 | `POST /v2/bot/richmenu` |
| 画像アップロード | `POST /v2/bot/richmenu/{id}/content` |
| ユーザーへ紐付け | `POST /v2/bot/user/{userId}/richmenu/{richMenuId}` |
| 紐付け解除 | `DELETE /v2/bot/user/{userId}/richmenu` |
| デフォルト設定 | `POST /v2/bot/user/all/richmenu/{richMenuId}` |

すべて無料。メッセージ送信数にカウントされない。

### 環境変数（wrangler.toml に追加予定）

```toml
[vars]
RICHMENU_ID_PATTERN1 = "richmenu-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
RICHMENU_ID_PATTERN2 = "richmenu-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
RICHMENU_ID_PATTERN3 = "richmenu-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
# RICHMENU_ID_PATTERN4 は将来追加
```

### 切り替えロジック（line_bot.ts への追加）

```typescript
// ユーザー連携完了時に呼び出す
async function assignRichMenu(userId: string, richMenuId: string, accessToken: string) {
  await fetch(`https://api.line.me/v2/bot/user/${userId}/richmenu/${richMenuId}`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${accessToken}` }
  });
}

// 連携解除時
async function removeRichMenu(userId: string, accessToken: string) {
  await fetch(`https://api.line.me/v2/bot/user/${userId}/richmenu`, {
    method: 'DELETE',
    headers: { Authorization: `Bearer ${accessToken}` }
  });
}
```

### 割り当てトリガー一覧

| イベント | 処理 |
|---|---|
| 新人社員の招待コード認証成功 | パターン1を割り当て |
| 班長・指導者の招待コード認証成功 | パターン2を割り当て |
| 車番連携のパスワード認証成功 | パターン2を割り当て |
| `れんけいかいじょ` 送信 | 紐付け解除（パターン3に戻る） |
| 一般乗務員の連携成功（将来） | パターン4を割り当て |

## 6. 備考・注意事項

- パターン3（デフォルト）はLINE OA Manager上で設定するため、APIでの明示的な割り当ては不要
- 同一ユーザーが複数の種別を持つ場合（例：指導者かつ車番検索管理者）は、どちらか一方を優先する（現状は指導者優先）
- リッチメニューIDはLINE OA Managerまたは `GET /v2/bot/richmenu/list` APIで取得可能
